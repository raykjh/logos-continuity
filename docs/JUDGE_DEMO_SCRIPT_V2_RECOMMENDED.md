# LOGOS Continuity — Recommended 2:35 Judge Demo Script V2

This version keeps the existing Judge Mode click order but sharpens the opening, reduces feature-list narration, and leaves approximately 25 seconds of safety margin under the three-minute limit.

## 00:00–00:15 — Problem and Product

**Action:** Show the main dashboard and the GPT-5.6 LIVE badge.

**Korean narration:**

AI는 이전 대화를 기억할 수 있습니다. 하지만 무엇이 지금의 확정된 사실인지는 보장하지 못합니다. LOGOS Continuity는 오래된 상태와 미확정 생각이 프로젝트 정본으로 섞이는 것을 막습니다.

**English subtitle:**

AI can remember previous conversations, but it cannot guarantee which information is the current confirmed truth. LOGOS Continuity prevents outdated state and uncertain ideas from silently becoming canonical project facts.

## 00:15–00:25 — Codex Collaboration

**Action:** Start Judge Mode.

**Korean narration:**

Codex와 함께 이 설계를 작동하는 공개 제품으로 구현했습니다. 상태 엔진, 승인 규칙, GPT 연동과 54개의 자동 테스트까지 완성했습니다.

**English subtitle:**

With Codex, I turned this design into a working public product with a state engine, approval rules, GPT integration, and 54 automated tests.

## 00:25–00:35 — Command Center

**Action:** Open Command Center and briefly show the health score and priority queue.

**Korean narration:**

Command Center는 현재 가장 위험한 연속성 문제를 찾아, 사용자가 해결해야 할 화면으로 안내합니다.

**English subtitle:**

Command Center identifies the most urgent continuity risks and routes the user to the correct resolution workflow.

## 00:35–01:00 — New Session Recovery

**Action:** Enter `LOGOS 해커톤 작업 이어가자` and show the Continuity Brief.

**Korean narration:**

새 세션에서 GPT-5.6이 자연어 요청으로 프로젝트를 식별합니다. LOGOS는 정본, 현재 상태, 다음 작업과 미검증 경고를 서로 분리해 복구합니다.

**English subtitle:**

In a new session, GPT-5.6 identifies the intended project from natural language. LOGOS restores canonical truth, current state, next actions, and unverified warnings as separate layers.

## 01:00–01:22 — Safe GPT-5.6 Routing

**Action:** Classify `유료화도 생각해보자.`

**Korean narration:**

GPT-5.6은 사용자의 의도를 구조화하지만 정본을 직접 바꾸지 못합니다. 이 문장은 Exploration에만 남고, 명시적 승인 전에는 Project Truth가 유지됩니다.

**English subtitle:**

GPT-5.6 structures the user's intent, but it cannot directly change canonical truth. This statement remains in Exploration until the user explicitly approves a change.

## 01:22–01:44 — Conflict Resolution

**Action:** Create the prepared conflicting proposal and show the three choices.

**Korean narration:**

기존 정본과 충돌하는 내용은 자동으로 덮어쓰지 않습니다. 사용자는 기존 결정을 유지하거나, 이번 작업만 예외로 적용하거나, 승인 후 새 정본으로 교체할 수 있습니다.

**English subtitle:**

Conflicting claims never silently overwrite canonical state. The user can keep the existing decision, apply a temporary exception, or approve a new canonical decision.

## 01:44–02:00 — History and Provenance

**Action:** Show State Operations and History.

**Korean narration:**

정본이 바뀔 때마다 무엇이 바뀌었는지, 왜 바뀌었는지, 어디에서 나온 정보인지 이력으로 남습니다.

**English subtitle:**

Whenever canonical state changes, LOGOS records what changed, why it changed, and where the information came from.

## 02:00–02:25 — Safe Handoff

**Action:** Open Assembled Context.

**Korean narration:**

그래서 다음 세션은 대화 전체를 다시 추측하지 않습니다. 검증된 현재 상태, 미해결 위험, 정확한 다음 행동을 권위 순서대로 전달받습니다.

**English subtitle:**

The next session does not have to guess from the full conversation again. It starts from verified project state, unresolved risks, and the correct next action in authority order.

## 02:25–02:35 — Close

**Korean narration:**

LOGOS는 장기 AI 작업을 추측 기반 회상에서 승인과 출처를 가진 연속성으로 바꿉니다.

**English subtitle:**

LOGOS turns long-running AI work from probabilistic recall into approval-gated, provenance-aware continuity.

## Recording rule

Do not fill the remaining time with extra features. Keep the final video near 2:35 and use the remaining margin for GPT response latency, clicking, and natural speech pauses.
