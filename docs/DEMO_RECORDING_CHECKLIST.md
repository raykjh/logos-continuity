# Demo Recording Checklist

Recording remains intentionally postponed until the entrant is ready. Everything below can be prepared in advance.

## Ready Before Recording

- [x] Public app deployed
- [x] GPT-5.6 LIVE verified
- [x] Public repository available
- [x] MIT license added
- [x] English README prepared
- [x] Devpost English draft prepared
- [x] Build Week evidence log prepared
- [x] Korean narration guide prepared
- [x] English subtitle file prepared
- [x] Seven-step Judge Mode rehearsed under three minutes
- [x] Browser warnings and errors checked

## User Action Before Recording

- [ ] Run `/feedback` in the primary Codex build thread
- [ ] Store the returned Session ID outside the video editor
- [ ] Choose the recording application
- [ ] Confirm microphone input and comfortable Korean narration
- [ ] Decide whether English subtitles will be burned in or uploaded to YouTube

## Recording Setup

- Use a 16:9 canvas, preferably 1920×1080.
- Record only the app window or a clean browser window.
- Hide notifications, bookmarks, personal tabs, API keys, and account details.
- Do not use copyrighted background music.
- Keep the public app open before starting the recording.
- Wake the Render instance and verify the GPT-5.6 LIVE badge.
- Click **데모 초기화** immediately before the final take.

## Required Video Content

- Product problem and target user
- Working public project
- Codex contribution
- GPT-5.6 runtime contribution
- One live project-recognition result
- One safe statement-classification result
- Conflict choices without silent overwrite
- Provenance or history evidence
- Final assembled context handoff

## After Recording

- Trim dead time without removing required explanation.
- Keep the final duration under three minutes.
- Add the complete English subtitles from `DEMO_SUBTITLES_EN.srt`.
- Check subtitle timing against the final edit.
- Confirm that no secrets or personal data appear.
- Upload publicly to YouTube.
- Test the YouTube URL in a signed-out browser.
- Add the URL to Devpost and the submission checklist.
